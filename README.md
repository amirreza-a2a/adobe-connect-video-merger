
# 🚀 Adobe Connect Video Merger (Dual-Engine)

این پروژه شامل مجموعه‌ای از اسکریپت‌های Bash و Python است که برای خودکارسازی فرآیند **اکسترکت، همگام‌سازی (Sync)، پر کردن گپ‌ها و ترکیب (Merge)** ویدیوها و فایل‌های صوتی تکه‌تکه شده در خروجی‌های زیپ Adobe Connect طراحی شده است.

این ابزار با دو موتور پردازشی مختلف (تایم‌لاین هوشمند و کپی سریع) به شما اجازه می‌دهد کلاس‌های آنلاین را با بالاترین دقتِ همگام‌سازی و بدون خطارهای رایجِ فایل‌های خراب، به یک ویدیوی یکپارچه تبدیل کنید.

---

## ✨ ویژگی‌های کلیدی

* **موتور تایم‌لاین جهانی (Timeline Engine):** محاسبه مبدأ زمان کلاس ($T_0$) بر اساس متادیتای XML تمام فایل‌ها به صورت میلی‌ثانیه‌ای.
* **الگوریتم پر کردن گپ‌ها (Gap Filling):** بازسازی خودکار ویدیو در زمان‌هایی که اساتید اسکرین‌شیر را قطع/وصل یا از وایت‌برد استفاده می‌کنند (تولید تصویر سیاه هوشمند جهت حفظ سینک صدا و تصویر).
* **پایپ‌لاین هوشمند صدا:** میکس خودکار تمام استریم‌های صوتی فعال (`cameraVoip`) و اعمال تاخیرهای متناسب، در کنار فیلتر کردن هوشمند فایل‌های صوتی خراب یا صفر بایتی.
* **سیستم دوگانه‌ی پردازش:** ارائه دو سناریوی متفاوت برای کلاس‌های عادی و کلاس‌های چندتکه‌ای با قابلیت انتخاب سرعت یا دقت بالا.

---

## 🛠 پیش‌نیازمندی‌ها

برای اجرای بدون نقص اسکریپت‌ها، ابزارهای زیر باید روی سیستم شما نصب باشند:

### لینوکس (اوبونتو)
* **FFmpeg & FFprobe**: جهت آنالیز متادیتا و رندر/میکس صدا و تصویر.
* **Unzip**: برای اکسترکت خودکار دسته‌ای فایل‌های فشرده.
* **Python 3**: برای پردازش منطق زمان‌بندی‌ها.

نصب در اوبونتو:
```bash
sudo apt update
sudo apt install ffmpeg unzip python3

```

### ویندوز (Windows)

* **Python 3**: دانلود و نصب از سایت رسمی (تیک گزینه‌ی `Add Python to PATH` در هنگام نصب حتماً زده شود).
* **FFmpeg**: دانلود نسخه ویندوز و اضافه کردن مسیر پوشه `bin` آن به Environment Variables سیستم (PATH) تا دستور `ffmpeg` و `ffprobe` در ترمینال ویندوز قابل شناسایی باشند.

## 📂 ساختار دایرکتوری‌ها و اسکریپت‌ها

اسکریپت‌ها در سیستم‌های لینوکسی به صورت پیش‌فرض بر اساس ساختار پوشه‌ای زیر کار می‌کنند:

```text
├── 📂 Downloads/
│   ├── 📂 adobeZip/       # محل قرارگیری فایل‌های زیپ دانلود شده از ادوبی
│   └── 📂 adobeVideo/     # محل ذخیره نهایی ویدیوهای ترکیب‌شده (.mp4 یا .mkv)
└── 📂 Scripts/
    ├── mergeVideoAdobe.py # اسکریپت اصلی (موتور جامع تایم‌لاین و رندر MP4)
    └── fast_merge.py      # اسکریپت فرعی (موتور کپی سریع بدون رندر مجدد - MKV)

```

> ⚠️ **نکته درباره Python Shebang (مخصوص لینوکس):** اسکریپت‌های پایتون به صورت پیش‌فرض از مفسر محیط مجازی در مسیر `~/Scripts/.scriptsENV/bin/python3` استفاده می‌کنند. در صورت نیاز می‌توانید خط اول فایل‌ها را متناسب با سیستم خود تغییر دهید. کاربران ویندوز نیازی به تغییر این خط ندارند و مفسر ویندوز آن را نادیده می‌گیرد.

## ⚙️ معرفی موتورهای پردازش

پروژه به شما دو ابزار با دو استراتژی متفاوت ارائه می‌دهد:

### ۱. موتور جامع تایم‌لاین (`mergeVideoAdobe.py`) - پیش‌فرض

* **کاربرد:** مخصوص کلاس‌هایی که استاد مدام اسکرین‌شیر را باز و بسته می‌کند، یا فایل صوتی خراب در پوشه وجود دارد.
* **مکانیزم:** تمام تکه‌های ویدیو را بر اساس زمان وقوع مرتب کرده، فضاهای خالی را با ویدیوی سیاه پر می‌کند و کل کلاس را به صورت فریم‌به‌فریم به فرمت استاندارد **MP4 (H.264)** رندر می‌کند. (تضمین ۱۰۰٪ سینک بودن صدا و تصویر).

### ۲. موتور کپی سریع (`fast_merge.py`)

* **کاربرد:** مخصوص کلاس‌های ایده‌آل که در آن‌ها کلاً یک فایل اسکرین‌شیر پیوسته و بدون قطع و وصل وجود دارد.
* **مکانیزم:** صداها را میکس کرده و بدون رندر مجدد تصویر، صرفاً استریم ویدیو را روی صدا کپی (`-c:v copy`) و خروجی **MKV** می‌دهد (انجام فرآیند در کمتر از چند ثانیه).

## 🚀 راهنمای اجرا

بسته به سیستم‌عامل خود، یکی از روش‌های زیر را برای اجرای اسکریپت انتخاب کنید:

### روش اول: در سیستم‌عامل لینوکس (اتوماسیون کامل و دسته‌ای)

در لینوکس به کمک اسکریپت بش نیازی به اکسترکت دستی ندارید. ابتدا دسترسی اجرا را اعطا کنید:

```bash
chmod +x process_adobe.sh
chmod +x ~/Scripts/mergeVideoAdobe.py
chmod +x ~/Scripts/fast_merge.py

```

سپس فایل‌های زیپ کلاس‌ها را در پوشه `~/Downloads/adobeZip` بریزید و اسکریپت اصلی را اجرا کنید:

```bash
./process_adobe.sh

```

### روش دوم: در سیستم‌عامل ویندوز (فرآیند دستی)

از آنجا که اسکریپت بش (`.sh`) به صورت بومی روی ویندوز قابل اجرا نیست، کاربران ویندوز باید فرآیند اکسترکت را به صورت دستی انجام دهند. از آنجا که اسکریپت پایتون فایل‌های `.flv` و `.xml` را در مسیر جاری خود (Current Directory) جستجو می‌کند، حتماً باید فرآیند زیر را طی کنید:

1. فایل زیپ کلاس ادوبی کانکت مورد نظر خود را به صورت عادی (مثلاً با WinRAR یا ۷Zip) درون یک پوشه کاملاً اکسترکت کنید.
2. وارد آن پوشه اکسترکت شده بشوید (جایی که فایل‌های `cameraVoip` و `screenshare` قرار دارند).
3. در فضای خالی پوشه، کلید `Shift` را نگه داشته و راست‌کلیک کنید، سپس گزینه‌ی **Open PowerShell window here** یا **Open Command Prompt here** را انتخاب کنید تا ترمینال دقیقاً در همان مسیر باز شود.
4. اکنون دستور اجرای اسکریپت پایتون را همراه با آدرس محل ذخیره اسکریپت وارد کنید.

به عنوان مثال اگر اسکریپت شما در درایو C ذخیره شده است، دستور زیر را در ترمینالِ پوشه کلاس وارد کنید:

```powershell
python "C:\path\to\mergeVideoAdobe.py"

```

*(اگر کلاس شما از نوع بدون گپ و ساده است، می‌توانید به جای آن `fast_merge.py` را صدا بزنید).*

پس از پایان پردازش، فایل ویدیویی نهایی با نام `final_class_synced.mp4` در همان پوشه در کنار بقیه فایل‌ها ساخته خواهد شد.
